**Category (Tiny Bubbles v2):** visual schedules · first-then · social stories · choice boards · token boards
**Date:** 2026-07-09
**Target stack:** Expo SDK 56 / React Native / TypeScript (Kid + Parent apps, Firebase backend)
**Scope note:** This category overlaps with AAC. Pure AAC infra repos (`openboardformat`, `OpenAAC-flutter`, `cboard-mobile`) are being handled by the sibling AAC agent; this inventory focuses on the **choice-board / board-builder / schedule** facet and the reusable data models behind it.

---

## TL;DR

The open-source landscape for *dedicated* visual-schedule / first-then / social-story / token-board apps is thin: the strong, mature, permissively-licensed projects are all **AAC symbol-board apps**, whose grid/board/category model is the same primitive we need for choice boards, first-then, and ordered visual schedules. Dedicated visual-schedule/first-then/social-story repos on GitHub are almost all **tiny hobby projects with no LICENSE** (all-rights-reserved) — reference-only at best. The best commercial products (Choiceworks, First Then Visual Schedule, Tiimo, Thruday) are closed-source.

**Cloned (MIT, shippable):**
1. `speakeasy-aac` — modern React 19 + TS + Zustand + Dexie board app; **best reusable board data model**; PRD explicitly scopes visual schedules.
2. `aac-native` (Leeloo AAC) — Expo/RN PECS choice-board app; multilingual symbol data + 158 bundled symbols + native TTS pattern.

**Reference-only (do NOT ship code):**
- `cboard-org/cboard` — **GPL-3.0** (copyleft). The gold-standard OSS AAC/board builder (740★, actively maintained). Study only.
- Several unlicensed dedicated repos (see §Reference-only).

## 1. Cloned repos (MIT — cleared to ship, code only)

### 1.1 speakeasy-aac  ✅ PRIMARY DONOR
- **Repo:** https://github.com/jeremydpotts/speakeasy-aac
- **Local:** `/Users/ameyakashid/Desktop/adhd india/_sources2/speakeasy-aac`
- **License:** **MIT** (`LICENSE`, "© 2024 SpeakEasy AAC Contributors") — verified in file and `package.json`.
- **Tech:** React 19 + **TypeScript** + Vite 7 + **Zustand** (state) + **Dexie** (IndexedDB, offline) + Tailwind v4 + vite-plugin-pwa. Web/PWA (not RN), but pure TS logic ports cleanly to Expo/RN.
- **Maturity:** Small (0★) but **recent** (pushed 2025-12) and unusually well-documented (`docs/PRD.md`, `ARCHITECTURE.md`, `RESEARCH.md`, `DEVELOPMENT_PLAN.md`).
- **What it is:** Symbol-based AAC board for nonverbal autistic children — core vocabulary, category navigation into sub-boards, TTS, customizable grid sizes, offline. PRD §1.2 explicitly names **"visual schedules"** as in-scope.
- **License caveat:** `public/symbols/*` are placeholder SVGs generated by `scripts/generatePlaceholderSymbols.js` (not a licensed symbol set) — safe, but they're placeholders, not production art.

**Reusable for Tiny Bubbles v2 — concrete pointers:**
- `src/types/index.ts` — **the board/symbol data model to adopt.** `Symbol` (`id, label, imageUrl, category, isCategory, isCore, backgroundColor`) and `Board` (`id, name, parentId, symbolIds[] (ordered), gridSize, sortOrder`). The ordered `symbolIds[]` + `parentId` tree is **directly reusable for choice boards, first-then (2 cells), and full visual schedules** — just constrain `gridSize`/count per use case. Also defines `GridSize` (`2x2`…`8x8`) and `Theme` (incl. `high-contrast`).
- `src/stores/boardStore.ts` — **board navigation engine (Zustand):** `navigationStack`, `navigateToCategory`, `goBack`, `goHome`, `canGoBack`. This is exactly the choice-board → sub-board drill-down. Port near-verbatim to a Firestore-backed Zustand store.
- `src/data/defaultData.ts` — seed content: core vocabulary set + `categoryToBoardMap`. Good scaffold for shipping default first-then/schedule/choice templates.
- `src/data/db.ts` — Dexie schema + `getBoardSymbols`. In v2 swap Dexie → Firestore (parent-authored boards) with an on-device cache; the query shape carries over.
- `src/components/SymbolGrid.tsx` + `SymbolCard.tsx` — responsive N×N grid with empty-cell fill and `GRID_DIMENSIONS` mapping. Reimplement in RN (`FlatList numColumns` / flex grid); logic is trivial to port.
- `src/services/speechService.ts` — TTS abstraction (Web Speech API). **Replace the engine with `expo-speech`** in RN; keep the rate/pitch/voice interface (`settingsStore.ts` already models `speechRate 0.5–2.0`, `speechPitch`, `showLabels`, `soundEffects`).
- `docs/PRD.md` / `docs/RESEARCH.md` — usable evidence-based requirements notes for our own feature specs.

### 1.2 aac-native (Leeloo AAC)
- **Repo:** https://github.com/btk/aac-native
- **Local:** `/Users/ameyakashid/Desktop/adhd india/_sources2/aac-native`
- **License:** **MIT** (`LICENSE`, "© 2020 Burak Tokak") — verified. Bundled font `fonts/OFL.txt` = SIL OFL 1.1 (Rubik).
- **Tech:** **Expo SDK 37 / React Native** (2020-era, class components, `react-navigation` v3, `expo-speech`, `expo-av`, `react-native-svg`, `react-native-storage`). 40★, last pushed 2023. **Stale SDK** — lift patterns/data/assets, not the app shell.
- **What it is:** Multilingual PECS-style AAC choice board (categories → cards, tap to speak, sentence "announcer" bar).

**Reusable for Tiny Bubbles v2 — concrete pointers:**
- `data/languages/card_*.json` (9 languages: en/de/el/es/fr/pa/pl/ro/tr) — **ready-made board content model:** `group` and `card` records with `title, slug, parents[], color, phrases`. The `parents[]` hierarchy = category/choice-board grouping; the per-group `color` is a good accessibility default. Reusable as seed data + a proven i18n content schema.
- `data/images/` — **158 bundled symbol PNGs** (feelings: afraid/angry/ashamed…, foods, people, activities). Useful as a starter symbol set. **License caveat:** repo code is MIT, but verify provenance of the symbol art before shipping (AAC symbol sets often carry their own terms, e.g. ARASAAC CC BY-NC-SA / Mulberry CC BY-SA); treat as reference until confirmed.
- `layouts/cards.js` / `layouts/groups.js` / `layouts/search.js` — grid/group rendering + filter-by-`parents` (`localizedCardData.filter(c => c.parents.includes(group))`) and symbol search. Pattern reference for our board renderer.
- `components/card.js` + `components/touchable-scale/` — tap-to-speak card with spring press-scale feedback (`API.speak(title)` + emits `announce`). Good kid-friendly tap UX pattern; `TouchableScale` → RN Reanimated in v2.
- `components/announcer.js` / `announcerButton.js` / `speaking.js` — **sentence-strip builder** (tapped symbols accumulate into an utterance, then speak). Directly relevant to choice-board/first-then output.
- `js/language.js` — light i18n runtime over `data/text.json` (5-language UI strings) — pattern reference for our accessibility-i18n spec.

## 4. Recommendation for Tiny Bubbles v2

1. **Adopt `speakeasy-aac`'s `Symbol`/`Board` TypeScript model** (`src/types/index.ts`) as the single primitive behind choice boards, first-then, and visual schedules — an ordered `symbolIds[]` + `parentId` tree covers all four. Back it with Firestore (parent authors boards in the Parent app; Kid app renders + caches).
2. **Port `boardStore.ts`** (navigation stack / drill-down) to a Firestore-backed Zustand store for choice-board category navigation.
3. **Reuse `aac-native`'s multilingual `card_*.json` content schema + i18n runtime** as the seed-content/localization pattern; feed our own cleared symbol art through it.
4. **TTS:** keep speakeasy's `speechService` interface, implement with `expo-speech`; keep aac-native's tap-to-speak + sentence-strip UX (`card.js` + `announcer.js`) reworked with Reanimated.
5. **Study `cboard` (GPL, no copy)** for the board-builder UX and Open Board Format import/export if we later want interoperability with existing AAC boards — but implement independently.
6. **Token boards** have no OSS donor — build natively (an ordered N-of-M reward strip is a thin variant of the same `Board` model + our existing gamification/reward engine from v1's `tether` port).
